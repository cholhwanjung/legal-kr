# /legal-kr:legal-risk-assessment-kr -- 한국법 위험도 평가

## Invocation

이 skill은 다음과 같은 상황에서 활성화됩니다:
- 사업 의사결정의 법적 위험을 체계적으로 평가할 때
- 규제 리스크 매핑이 필요할 때
- 에스컬레이션 여부를 판단할 때
- `/legal-kr:legal-risk-assessment-kr` 슬래시 커맨드가 호출될 때

## Workflow

### Step 1: 평가 대상 수신

사용자로부터 평가 대상을 수신합니다:
- 계약/거래 구조
- 신규 사업 모델
- 규제 변경에 따른 영향

### Step 2: 컨텍스트 수집

- **산업 분야**: IT/SaaS, 금융, 제조, 유통 등
- **거래 규모**: 연간 거래금액, 대상 고객 수
- **당사자 관계**: B2B 대등 / B2B 원·하도급 / B2C 소비자
- **규제 환경**: 특별법 적용 여부 (금융, 의료, 교육 등)

### Step 3: Risk Matrix 구성

**발생 가능성 (Likelihood)**:
| 등급 | 기준 |
|------|------|
| 높음 | 유사 사례에서 규제 조치/소송 발생 이력 있음. 관련 판례 존재. |
| 중간 | 법령 해석에 따라 위반 소지 있으나 명확한 판례 없음. |
| 낮음 | 이론적 위험만 존재. 실제 집행 사례 없음. |

**영향도 (Impact)**:
| 등급 | 기준 |
|------|------|
| 높음 | 계약 무효, 과징금, 형사 처벌, 사업 중단 리스크 |
| 중간 | 계약 일부 무효, 시정 명령, 손해배상 |
| 낮음 | 경고, 권고, 계약 수정으로 해결 가능 |

### Step 4: 한국법 규제 환경 분석

주요 규제기관 및 적용 법령:

| 규제기관 | 관할 영역 | 주요 제재 |
|----------|-----------|-----------|
| **공정거래위원회** | 약관규제, 공정거래, 하도급 | 시정명령, 과징금, 형사고발 |
| **금융감독원** | 금융거래, 전자금융 | 기관제재, 과태료, 인허가 취소 |
| **개인정보보호위원회** | 개인정보 처리 | 과징금(매출 3%), 시정명령 |
| **방송통신위원회** | 정보통신, 온라인 서비스 | 과징금, 서비스 중지 |
| **고용노동부** | 고용, 근로 조건 | 시정명령, 과태료, 형사처벌 |

### Step 5: 위험 항목별 평가

각 식별된 위험에 대해:
1. **위험 설명**: 구체적 시나리오
2. **관련 법령**: 적용 조항
3. **발생 가능성**: 높음/중간/낮음 + 근거
4. **영향도**: 높음/중간/낮음 + 예상 제재
5. **위험 등급**: RED/YELLOW/GREEN (가능성 x 영향도)
6. **완화 방안**: 구체적 조치

### Step 6: 에스컬레이션 판단

**즉시 에스컬레이션 (외부 자문 필요)**:
- 강행법규 위반이 명백한 경우
- 과징금/형사 처벌 리스크가 있는 경우
- 규제기관 조사/심결 진행 중인 경우

**내부 검토 충분**:
- 계약 수정으로 해결 가능한 경우
- 위험이 이론적 수준인 경우

### Step 7: 모니터링 권장

법령 변경 모니터링 대상 제시:
- 법제처 입법예고 (법제처 웹사이트)
- 국회 계류 법안 (의안정보시스템)
- 규제기관 고시/가이드라인 변경

## Notes

- 이 평가는 법률 자문을 대체하지 않습니다. RED 등급 위험은 반드시 자격을 갖춘 법률 전문가의 검토를 거쳐야 합니다.
- 한국의 규제 환경은 빠르게 변화합니다. 평가 시점의 법령을 기준으로 하며, 최신 입법 동향을 별도로 확인하십시오.
- 특정 조항의 심층 분석이 필요하면 `/legal-kr:adversarial-review`를 활용하십시오.
